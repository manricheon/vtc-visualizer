# VTC Visualizer

CSV/JSON 데이터를 브라우저에서 논문·기술문서 스타일의 인터랙티브 그래프로 그리는 범용 시각화 도구입니다.
토큰 버짓 대비 성능, 속도 대비 성능 같은 벤치마크 비교 그래프를 자유롭게 커스텀할 수 있습니다.
모든 데이터는 브라우저 안에서만 처리되며 외부로 전송되지 않습니다.

## 빠른 시작

세 가지 방법 중 아무거나:

1. **더블클릭** — `index.html`을 브라우저로 열고 파일을 드래그 앤 드랍 (인터넷 필요: 차트 라이브러리를 CDN에서 로드)
2. **완전 오프라인** — `index-offline.html`을 더블클릭 (인터넷 불필요, 파일 하나만 복사해가면 어디서든 동작)
3. **폴더 자동 로드** — 데이터 폴더를 지정해 서버 실행:

   ```bash
   python visualizer.py .                 # 처음이라면: 동봉된 example.csv로 바로 체험
   python visualizer.py results/          # results/ 안의 *.csv, *.json 자동 로드
   python visualizer.py                   # 빈 상태로 시작
   python visualizer.py results/ --port 8765
   ```

동봉된 `example.csv`를 드래그해서 기능을 먼저 둘러봐도 좋습니다 — 4개 method × 토큰 버짓(500~16k) 스윕 24행(중복 없음)으로,
토큰이 많을수록 정확도는 포화 곡선으로 오르고 지연·비용은 늘어나는 식의 상식적인 추세가 들어 있습니다
(예: X=tokens, Y=accuracy, 그룹=method — 필터 없이 바로 깔끔한 곡선이 나옵니다).

## 데이터 입력 방법

- CSV/JSON 파일 **드래그 앤 드랍** (여러 개 동시 가능)
- **파일 열기… / 폴더 열기…** 버튼
- **직접 입력…** 버튼 → CSV/JSON 텍스트 붙여넣기
- `visualizer.py`로 실행한 경우 지정 폴더의 파일 자동 로드

파일을 계속 추가하면 데이터가 누적 병합됩니다. 같은 파일명을 다시 넣으면 교체됩니다.

## 데이터 포맷 (입력 계약)

- **CSV**: 첫 줄이 헤더, 이후 한 행 = 한 측정점. (TSV도 지원)
- **JSON**: 객체 배열 `[{"method": "ours", "tokens": 4000, "score": 0.744}, …]`
- **필수 컬럼은 없습니다.** 숫자 컬럼은 자동으로 축 후보가 되고, 문자열 컬럼은 그룹(색상)·필터 후보가 됩니다.
- 컬럼 이름은 자유이며, 파일마다 컬럼이 달라도 됩니다(합집합으로 병합, 빈 값은 `–`).
- 파일을 2개 이상 로드하면 출처 파일명이 `_source` 컬럼으로 나타나 그룹·필터에 쓸 수 있습니다 (1개일 땐 숨겨짐).

권장 형태 (long-form / tidy — 한 행에 한 측정점):

```csv
method,tokens,latency_s,score,dataset
baseline,1000,1.2,0.612,MMLU
baseline,4000,3.8,0.681,MMLU
ours,1000,1.4,0.641,MMLU
ours,4000,4.1,0.744,MMLU
```

## 주요 기능

| 기능 | 사용법 |
|---|---|
| 차트 추가/복제/삭제 | 상단 `＋ 차트 추가`, 카드의 `복제`/`삭제` — 한 페이지에 여러 그래프 |
| **그래프 조작** | 드래그=영역 확대 · 휠=줌 · 더블클릭=전체 보기 · 팬은 모드바 십자 화살표 · `보기 초기화` 버튼으로 리셋. 줌 상태는 스타일을 바꿔도 유지됨 |
| 축·스케일 | 설정 → 축: 레이블, linear/log 토글, min/max 범위(**한쪽만 입력해도 됨**), 그리드 |
| 시리즈 스타일 | 설정 → 스타일: 시리즈별 색·마커 심볼/크기·**선 종류(실선/파선/점선)·선 굵기**, 폰트, 범례 위치 |
| 필터 | 설정 → 필터: 컬럼 선택 → 카테고리면 **값 체크박스(여러 개 선택 가능** — 예: method에서 baseline·ensemble 두 개만 체크), 숫자면 비교 연산(>, ≥ 등). 조건을 모두 만족하는 행만 그려짐 |
| **언어 (KO/EN)** | 우상단 토글 버튼으로 한국어 ↔ English 전환 (선택 저장됨) |
| **베이스라인** | 포인트 **클릭** → 메뉴에서 "베이스라인 추가" → 가로/세로 점선. **여러 개 지정 가능**, 각각 사분면 음영 선택, 설정 패널에서 개별 삭제 |
| **텍스트 마커** | 포인트 **클릭** → "텍스트 마커 추가" → 화살표 달린 주석. 드래그로 이동, 클릭으로 수정/삭제 |
| **포인트 제외** | 포인트 **클릭** → "이 포인트 제외" → 모든 차트에서 제거. 토스트의 `실행 취소`, 표의 체크박스, `제외 N행 모두 복원` 버튼으로 롤백 |
| **추세선** | 설정 → 고급: 선형·2차 다항·로그·지수·거듭제곱·이동평균 피팅을 시리즈별로 표시 (선 종류·굵기 조절), **오차 밴드(±1σ/±2σ)** 음영 표시 가능 |
| **모양 그룹 (3번째 차원)** | 설정 → 데이터 → 모양 그룹: 색=그룹1, **마커 모양=그룹2** 이중 인코딩. 예: 색=method, 모양=frames → method별 색은 유지하면서 frames별 경향을 모양으로 구분 |
| 텍스트 마커 스타일 | 포인트 레이블 그룹에서 전체 글자 크기/색/배경/화살표 설정, 마커 클릭 팝업에서 개별 색·크기 오버라이드 |
| 선 스무딩 | 설정 → 스타일 → 선 종류: 직선/곡선(spline), 실선/파선/점선 |
| **영역 채우기** | 설정 → 스타일 → 영역 채우기: 선 아래를 시리즈 색의 연한 파스텔톤으로 채움 |
| 포인트 레이블 | 설정 → 포인트 레이블: 표시 후 **드래그**로 위치 조정, **클릭**으로 개별 숨김. 중복 값은 하나만 표시되고 겹침은 자동 회피 |
| Pareto frontier | 설정 → 고급: 방향(예: X작을수록·Y클수록 좋음) 선택 |
| 내보내기 | 카드의 `PNG`(3배 해상도) / `SVG` 버튼 |
| 원본 데이터 | 하단 표: 검색, 컬럼 클릭 정렬, 데이터셋별 삭제, **행 앞 체크박스 해제로 그래프에서 제외** |
| 세션 | 자동 저장(localStorage) + `세션 내보내기/가져오기`(JSON 파일)로 공유 가능 |

### 팁: 세 번째 차원 함께 보기

X=tokens, Y=accuracy, 그룹(색)=method로 그린 상태에서 frames 같은 추가 차원의 경향도 보려면:
1. **모양 그룹**에 frames 지정 — 색은 method, 마커 모양은 frames (가장 권장, 범례에 조합별 표시)
2. **크기 컬럼**에 frames 지정 — frames가 클수록 마커가 커짐 (연속값에 적합)
3. **포인트 레이블** 컬럼을 frames로 — 각 점 옆에 frames 값 표시
4. 차트 **복제** 후 필터로 frames 값별 분리 — 나란히 비교 (small multiples)

## 내 데이터를 이 포맷으로 변환하기 (에이전트 요청문)

자기 로그/실험 결과의 포맷이 무엇이든, 아래 요청문을 Claude 등 아무 LLM 에이전트에 복사해서 붙여넣고
자신의 데이터 파일(또는 경로)을 함께 주면 됩니다:

```text
아래 "입력 계약"에 맞는 CSV 또는 JSON 파일로 내 데이터를 변환해줘.

[입력 계약 — VTC Visualizer]
1. CSV(첫 줄 헤더) 또는 JSON 객체 배열. 인코딩 UTF-8.
2. long-form(tidy) 형태: 한 행 = 한 측정점. 반복 측정이면 행을 반복한다.
   (예: method별 × token budget별 점수라면 method,tokens,score 3컬럼으로 행을 나열)
3. 필수 컬럼은 없다. 다만 다음을 지키면 좋다:
   - 비교 대상(방법/모델/설정 이름)은 문자열 컬럼 하나로 (예: method) → 그래프의 색상 그룹이 된다.
   - 축으로 쓸 값(토큰 수, 시간, 점수 등)은 각각 숫자 컬럼으로. 단위는 컬럼명에 명시 (예: latency_s, cost_usd).
   - 조건 구분(데이터셋, GPU 종류 등)도 각각 별도 컬럼으로 → 필터로 쓸 수 있다.
4. 숫자 컬럼에는 숫자만 넣는다 (단위 문자열 금지, 천단위 콤마 금지, 결측은 빈 값).
5. 컬럼명은 영문 소문자+언더스코어 권장. "_source"라는 이름은 예약되어 있으니 쓰지 말 것.

변환 결과는 *.csv 또는 *.json 파일로 저장해줘. 변환 규칙이 애매한 부분은 나에게 물어봐줘.
```

변환이 자주 필요하면 위 계약에 맞는 자기만의 변환 스크립트를 만들어 쓰는 것도 좋습니다.

## 오프라인 버전 재생성

`index.html`을 수정했다면 오프라인 버전을 다시 만들어야 합니다:

```bash
python visualizer.py build-offline    # → index-offline.html (~4.6MB)
```

## English

The UI is fully bilingual — click the **EN** button in the top-right corner to switch the entire interface to English
(the choice is remembered). Quick start: open `index.html` in a browser (or `index-offline.html` for air-gapped machines,
or `python visualizer.py <data-dir>`), then drag & drop CSV/JSON files. See the input contract above — any tidy table works.

## 요구 사항

- 브라우저 (Chrome/Edge/Safari/Firefox 최신)
- `visualizer.py` 사용 시 Python 3.8+ (표준 라이브러리만 사용, 별도 설치 불필요)
